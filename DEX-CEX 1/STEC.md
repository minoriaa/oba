## Часть 1: Что такое DEX-CEX арбитраж

### Суть

text

```
DEX (Децентрализованная биржа)     CEX (Централизованная биржа)
├── PancakeSwap (BSC)              ├── Bybit
├── Raydium (Solana)               ├── OKX
├── Jupiter (Solana)               ├── Gate
└── BiSwap (BSC)                   ├── KuCoin
                                   ├── MEXC
                                   └── HTX

Арбитраж = один и тот же токен стоит по-разному
на DEX и CEX → покупаем дешевле, продаём дороже
```

### Два направления сделки

text

```
Направление 1: DEX → CEX
┌──────────┐    Вывод     ┌──────────┐
│ Купил на │───токена────→│ Продал на│
│ DEX      │   (5-30мин)  │ CEX      │
│ $0.95    │              │ $1.00    │
└──────────┘              └──────────┘
Прибыль: 5% минус комиссии

Направление 2: CEX → DEX
┌──────────┐    Вывод     ┌──────────┐
│ Купил на │───токена────→│ Продал на│
│ CEX      │   (5-30мин)  │ DEX      │
│ $0.95    │              │ $1.00    │
└──────────┘              └──────────┘
Прибыль: 5% минус комиссии
```

### Почему BSC для старта

text

```
✅ Самый низкий порог входа
├── EVM-совместимость (знакомые инструменты)
├── Web3.py, ethers.js — всё работает
├── Тонны документации и примеров
└── Можно собрать прототип за выходные

✅ Оптимальный баланс цена/скорость
├── Газ достаточно низкий ($0.10-0.30 за свап)
├── 3 секунды блок — приемлемо для CEX-DEX
└── Хорошая ликвидность на PancakeSwap

✅ Нативная связь с Binance
├── Быстрый вывод BEP-20 токенов
├── Низкие withdrawal fees
└── Все 6 целевых бирж поддерживают BSC
```

### Почему Solana как второй этап

text

```
✅ Преимущества
├── Самый дешёвый газ ($0.001-0.01)
├── Самые быстрые блоки (0.4 сек)
├── Огромная экосистема мемкоинов (больше возможностей)
├── Jupiter — лучший DEX агрегатор в крипто
└── Много токенов с высокой волатильностью

⚠️ Сложности
├── Новая парадигма (аккаунтная модель vs EVM)
├── Другие инструменты (не Web3.py)
├── Нода дорогая ($400-1000/мес)
├── gRPC вместо WebSocket для реалтайма
└── Конкуренция MEV-ботов очень высокая

✅ Упрощение для CEX-DEX
├── Jupiter API снимает 80% сложности
├── Rust НЕ нужен (Python/Go достаточно)
├── Своя нода НЕ нужна (Helius $49/мес)
└── Jito bundles НЕ нужны на старте
```

### Поток данных

text

```

    BSC нода ──WebSocket──→ Sync event ──→ Новые резервы ──→ Цена DEX
    Jupiter API ──HTTP──→ Price response ──→ Цена DEX

    Для каждого токена:
    ├── DEX цена vs CEX bid (DEX→CEX направление)

    Спред > 0.5%?
    ├── ДА → Проверить ликвидность

    │        │                 ├── Открыт? → АЛЕРТ / ИСПОЛНЕНИЕ
    │        │                 └── Закрыт? → Пропустить
    │        └── Мало? → Пропустить
    └── НЕТ → Пропустить

Шаг 4: Исполнение (автоматическое, будущее)

    DEX→CEX:
    ├── Отправить swap TX на DEX
    ├── Дождаться получения токена
    ├── Депозит на CEX
    ├── Дождаться зачисления
    └── Продать на CEX

    Или гибрид (балансы на обеих сторонах):
    ├── Одновременно: Buy DEX + Sell CEX
    └── Потом ребалансировка
```

## Часть 6: Технологии чтения DEX цен

### BSC: чтение цен через ноду

text

```
Метод 1: Прямое чтение резервов пула

    Контракт PancakeSwap Pair
    └── getReserves() → reserve0, reserve1
        └── price = reserve1 / reserve0 (с учётом decimals)

    Один вызов = одна цена
    Задержка: ~5мс (своя нода) / ~50мс (провайдер)

Метод 2: Multicall (батч запросы)

    Один вызов Multicall контракта
    └── 500 пар getReserves() за раз
        └── 500 цен за ~100мс

    Вместо 500 отдельных запросов (~50 секунд)
    Ускорение: 500x

Метод 3: WebSocket подписка на Sync события

    Подписка на событие Sync(uint112, uint112)
    └── Каждый свап обновляет резервы
        └── Мы получаем новую цену мгновенно
            └── Сравниваем с CEX в реалтайме

    Задержка: 1 блок (3 секунды) + пропагация
```

### Solana: чтение цен через Jupiter

text

```
Метод 1: Jupiter Price API

    GET https://price.jup.ag/v6/price?ids={mints}
    └── Цены всех токенов за один запрос
        └── Обновляются каждые ~1-3 секунды

    Простота: максимальная
    Задержка: 200-500мс

Метод 2: Jupiter Quote API

    GET https://quote-api.jup.ag/v6/quote
    └── Точная котировка с учётом:
        ├── Маршрутизации через лучшие пулы
        ├── Slippage
        ├── Price impact
        └── Комиссий

    Точность: максимальная
    Задержка: 300-800мс

Метод 3: Прямое чтение пулов (продвинутый)

    RPC вызов getAccountInfo()
    └── Читаем аккаунт пула Raydium/Orca
        └── Декодируем резервы из байтов
            └── Считаем цену

    Сложность: высокая (аккаунтная модель)
    Задержка: 50-200мс
```

## Часть 7: MEV и безопасность транзакций

### Что такое MEV

text

```
MEV = Maximal Extractable Value
(Максимальная извлекаемая ценность)

Это прибыль, которую валидаторы/боты получают
за счёт переупорядочивания транзакций в блоке.

Пример: Sandwich Attack

    1. Ты отправляешь: "Купить 1000 CAKE"
    2. MEV бот ВИДИТ твою TX в mempool
    3. Бот ставит свою TX ПЕРЕД твоей (покупает CAKE → цена растёт)
    4. Твоя TX исполняется по ХУДШЕЙ цене
    5. Бот ставит TX ПОСЛЕ твоей (продаёт CAKE → забирает разницу)

    Твоя потеря: 0.5-3%
    Прибыль бота: твоя потеря
```

### Защита от MEV

text


## Часть 8: Сканирование токенов

### Как найти все арбитражные пары

text

```
Шаг 1: Получить список токенов на CEX (все 6 бирж)

    Bybit:  GET /v5/market/tickers → 500+ пар
    OKX:    GET /api/v5/market/tickers → 600+ пар
    Gate:   GET /api/v4/spot/tickers → 1500+ пар
    KuCoin: GET /api/v1/market/allTickers → 800+ пар
    MEXC:   GET /api/v3/ticker/24hr → 2000+ пар
    HTX:    GET /market/tickers → 600+ пар

    Объединяем → уникальные символы → 2000-3000 токенов

Шаг 2: Найти эти токены на DEX

    BSC (PancakeSwap):
    ├── Factory.allPairs() → перебираем все пулы
    ├── Для каждого пула: token0.symbol(), token1.symbol()
    ├── Сопоставляем с CEX списком
    └── Результат: 200-500 пар торгуются и на DEX и на CEX

    Solana (Jupiter):
    ├── Jupiter Token List API → все торгуемые токены
    ├── Сопоставляем с CEX списком
    └── Результат: 100-300 пар

Шаг 3: Фильтрация по ликвидности

    Отсекаем пулы с ликвидностью < $10,000
    ├── Мало ликвидности = большой slippage
    ├── Мало ликвидности = нельзя торговать объёмом
    └── Оставляем только жизнеспособные пары

Шаг 4: Мониторинг в реалтайме

    Для каждой найденной пары:
    ├── DEX: подписка на обновления цен
    ├── CEX: подписка на orderbook (все 6 бирж)
    └── Сравнение при каждом обновлении
```
